TSGDS are built to ensure minimum conflicts and any reduction of performance when stack with other frameworks,for your case will be bootstrap.

To further minimise the conflict, these are some important points you need to take note when using both SGDS and Bootstrap framework:
  1. Always include the SGDS.css after Bootstrap.css in the tag. This is to ensure the SGDS stylesheet takes priority over bootstrap for the mandatory components styles (masthead, main navigation, side navigation & footer).
  2. As much as possible, try to keep the same html structure to the mandatory components as they may be updated if there is amendment to the DSS.
  3. Use only SGDS flexbox grid system. In order to do this, you need to use the SASS import method.

You can also read up on the documentation on how to use SASS import here:

  • Customise SGDS with Sass
  • Bootstrap Sass customisation

This method can also be used to remove unused components and can reduce the file size and loading time for both CSS. Spacing (paddings and margins) and typography(font sizes) for the components are different in both frameworks. You might need to tweak the styling according to your site design.
