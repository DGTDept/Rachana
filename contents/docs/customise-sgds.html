If you use Sass in your project, you can use your own custom Sass source in conjunction with SGDS to achieve a high level of customisation.

File Structure

You’ll have a file structure that looks like this if using a package manager like npm:

{%- highlight bash -%} site-folder/ ├── scss │ └── custom.scss └── node_modules/ └── sgds-govtech/sass/ └── sgds.scss {%- endhighlight -%}

If you’ve manually downloaded our source files and aren’t using a package manager, you’ll want to setup something similar to that structure.

{%- highlight bash -%} site-folder/ ├── scss │ └── custom.scss └── sgds-govtech/sass/ └── sgds.scss {%- endhighlight -%}
Importing SGDS.scss

You can import the full SGDS framework

{% highlight scss %} // Version 1.0.24 @charset "utf-8"; // Custom sass files @import 'custom.scss'; $primary:black; $secondary:red; $warning:red; $button-color: yellow; // Required sass files @import "../node_modules/sgds-govtech/sgds/sass/sgds.scss {% endhighlight%}

or pick the SGDS components needed for your project

{% highlight scss %} @charset "utf-8"; // Custom sass files @import 'custom.scss'; $primary:black; $secondary:red; $warning:red; $button-color: yellow; // Required sass files @import "../node_modules/sgds-govtech/sgds/sass/sgds-customise/all"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-base/all"; // Optional @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-masthead"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-navbar"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-side-navigation"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-footer"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-button"; @import "../node_modules/sgds-govtech/sgds/sass/sgds-components/sgds-accordion"; {% endhighlight%}
