Guidelines

Card should:

  • Use headings that set clear expectations about its purpose
  • Prioritise information of the content so user know what comes first
  • Avoid too many call-to-action buttons and only one primary call-to-action per card

Style

Selector Description
sgds-card the main container
sgds-card-header a horizontal bar with a shadow
sgds-card-header-title a left-aligned bold text
sgds-card-content a multi-purpose container for any other element
sgds-card-footer a horizontal list of controls
sgds-card-footer-item a repeatable list item
